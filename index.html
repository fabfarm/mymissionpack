Seeed Studio Mission Pack - Documentation

Welcome to the documentation hub for the Seeed Studio Mission Pack, deployed on the Algarve FabFarm infrastructure. This documentation serves as a detailed guide to understanding, configuring, and utilizing the services and tools provided within the Mission Pack.

The Hazard Response Mission Pack is an open-source AIoT solution designed to efficiently manage and mitigate hazard scenarios. This compact system combines a range of technologies, including sensor networks and edge computing, to enable effective remote sensing, data transmission, processing, inference, and analysis. For more detailed information about the Hazard Response Mission Pack, please refer to the official GitHub repository.

Below, you'll find links to detailed documentation for each service included in the Mission Pack:

  • Interfaces and Communication - Overview of hardware and network interfaces.
  • SSH Server - Securely access the server terminal remotely.
  • CUPS Scheduler - Manage and configure print jobs.
  • Node-RED - Visual programming for IoT workflows.
  • MQTT Broker - Message broker for IoT communication.
  • PostgreSQL Database - Database for data storage and management.
  • LoRa Gateway Bridge - Connect LoRa devices to the network.
  • ChirpStack Application Server - Manage LoRaWAN networks and applications.
  • Media Server - Stream and manage media files.
  • Supervisor Service - Monitor and control server processes.
  • General Information - Overview of the Mission Pack and its components.
  • To do - List of tasks to be completed.
